# Проект "Графический редактор"

## О проекте

Графический редактор с веб-интерфейсом.

## Функциональные требования

- Должна быть возможность:

  - Создавать новый холст и рисовать на нем

  - Работать со слоями:

    - Создание

    - Изменение порядка наложения друг на друга (перемещение)

    - Скрытие

    - Изменение прозрачности

  - Отменять действия и возвращать их обратно (сохранение истории действий);

  - Экспортировать рисунок (в PNG).

- Должны быть реализованы следующие инструменты:

  - “Кисть” — рисует произвольные линии;

  - “Ластик” — стирает нарисованное;

  - “Линия” — рисует прямые линии;

  - “Фигуры” — создает квадрат или круг.

- При использовании инструментов “Ластик”, “Линия”, “Кисть”, “Фигуры” должна быть возможность выбирать толщину из 6
  вариантов.

- При использовании инструментов “Линия”, “Кисть”, “Фигуры” должна быть возможность выбрать цвет.

- При использовании инструмента “Фигуры” должна быть возможность выбора цвета обводки и внутренней части фигуры.

- При рисовании, курсор должен меняться на соответствующую иконку инструмента.

## Нефункциональные требования

- Приложение должно иметь хорошую производительность и быстрый отклик.

- Приложение должно быть легко масштабируемым и поддерживаемым.

## Технические требования

- Код должен быть написан на TypeScript.

- Приложение должно быть реализовано с использованием React.

- В приложении должен использоваться любой менеджер состояния (например, Redux или MobX).

- Приложение должно быть по максимуму покрыто тестами с использованием Jest, Testing Library. Рекомендуется вести
  разработку через тестирование.

## Пояснения

- Для сборки может использоваться любой из инструментов (например, Webpack или Vite).

- Для стилизации можно использовать любой подход: простой CSS, препроцессоры (Sass / Less и т. д.) или библиотеки
  компонентов (MUI, Ant Design и т. д.).

- Команда самостоятельно определяет общие договоренности по архитектуре проекта (файловая структура, использование
  методологий (например, FSD) и так далее).

- Работа должна проходить через Git по одной из выбранных методологий (например, Git Flow). Версии необходимо называть в
  соответствии со спецификацией[семантического версионирования](https://semver.org/).

### 2. Редактор

**Адрес:** /projects/:projectId/

**Содержание и поведение:**

- Верхнее меню

  - “Файл”

    - “Новый проект” — появляется модальное окно

      - Содержание:

        - Поле ввода — название проекта

        - Поле ввода — ширина (в пикселах)

        - Поле ввода — высота (в пикселах)

        - Кнопка “Создать”

      - Поведение

        - Окно появляется поверх содержимого страницы, фон затемнен

        - При нажатии на затемненный фон модальное окно закрывается

        - При нажатии на кнопку “Создать” текущий проект сохраняется и происходит переход на страницу
          /projects/:projectId/ (по новому идентификатору проекта), где будет открыт холст необходимого размера.

    - “Сохранить” — сохранение текущих изменений проекта. При повторном открытии страницы проекта загрузятся изменения
      из последнего сохранения.

    - “Экспортировать” — экспорт рисунка в формате PNG.

  - “Правка”

    - “Отменить” — отменяет последнее действие

    - “Вернуть” — повторяет последнюю операцию

  - “Все проекты” — переход на главную страницу

- Меню слева

  - “Инструменты”:

    - “Кисть” — рисует произвольные линии

    - “Ластик” — стирает нарисованное

    - “Линия” — рисует прямые линии

    - “Фигуры” — создает квадрат или круг (пользователь выбирает)

  - “Выбор цвета” — при нажатии появляется палитра с возможностью выбора активного цвета (будет использован при работе с
    инструментами “Кисть”, “Линия”, “Фигуры”)

- Меню справа

  - “Слои”

    - Кнопка “Создать слой” в виде иконки знака плюс

    - Ползунок, указывающий значение непрозрачности для выбранного слоя

    - Список слоев с прокруткой. Каждый элемент:

      - Название

      - Кнопка “Скрыть” в виде иконки глаза. В случае если элемент скрыт, глаз перечеркнут.

      - Кнопка “Настройки” в виде иконки, состоящей из трех точек. При нажатии появляется дополнительное меню с
        элементами:

        - Удалить — при нажатии удаляет слой

        - Переименовать — при нажатии появляется возможность изменить название данному слою

  - “История”

    - Список элементов, где каждый элемент — это одно совершенное действие

    - При нажатии на элемент происходит откат к выбранному действию. Элементы, которые были после выбранного действия
      становятся полупрозрачными, при нажатии на них история восстанавливается

- Рабочая область (место, где происходит рисование)
